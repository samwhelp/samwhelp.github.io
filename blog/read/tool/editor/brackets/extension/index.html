Brackets Extension

測試環境

以下測試環境在

  • 「Xubuntu 14.04 64位元」
  • 「Brackets.Release.0.44.64-bit」

Ubuntu底下的安裝方式，請參考「這篇」。

Extension Manager Shortkey

  • GitHub

安裝方式

  • 功能選單「File / Extension Manager」，呼叫出「Extension Manager」。在搜尋控制項輸入「short」。就會看到一筆「Extension Manager Shortkey」。按下那筆右方的「install」按鈕安裝。

使用方式

  • 「快捷鍵」是，「Shift+Alt+O」。就可以直接開啟「Extension Manager」對話框。

Brackets Commands Quick Search

  • GitHub

安裝方式

  • 開啟「Extension Manager」對話框。在搜尋控制項輸入「guide」。就會看到一筆「Brackets Commands Quick Search」。按下那筆右方的「install」按鈕安裝。

使用方式

*「快捷鍵」是，「Ctrl+Alt+/」。就會出現一個panel在上方，有個「?」。然後就可以打「關鍵字」，功能有點像「Sublime Text」的「Ctrl+Shift+P」。你可以打「關鍵字」，就會出現相關的「Command」，供你選擇執行。

  • 快捷鍵「Ctrl+Shift+O」，啟動「Quick Open」，然後輸入「?」。就會跟上面的結果一樣了。

修改快捷鍵

為了避免跟下面的「Display Shortcuts」這個「Plugin」的快捷鍵衝突。
要把快捷鍵改成「Ctrl+Shift+P」，除了可以跟「Sublime Text」習慣一樣。
也可以跟「Quick Open」的快捷鍵「Ctrl+Shift+O」，相呼應。

  • 開啟「~/.config/Brackets/extensions/user/pflynn.brackets.commands.guide/main.js」這個檔。

Display Shortcuts

  • GitHub

安裝方式

  • 開啟「Extension Manager」對話框。在搜尋控制項輸入「shortcuts」。就會看到一筆「Display Shortcuts」。按下那筆右方的「install」按鈕安裝。

使用方式

*「快捷鍵」是，「Ctrl+Alt+/」。不過這個會跟「Brackets Commands Quick Search」相衝。(請參考上面怎麼改快捷鍵)
就會看到下方有一個列表，有列出來一些快捷鍵。

  • 可以在「Filter」那輸入關鍵字查詢，例如:輸入「quick」。
    就可以看到四筆快捷鍵的資料。

Markdown Preview

  • GitHub
  • Article
  • YouTube

安裝方式

  • 開啟「Extension Manager」對話框。在搜尋控制項輸入「markdown」。就會看到一筆「Markdown Preview」。按下那筆右方的「install」按鈕安裝。

使用方式

  • 當開啟的檔案，副檔名是「.md」或「.markdown」。在右方的功能鍵，就會出現一個「M」和向下箭頭的圖示，按下它就會出現「Live Preview」在編輯區域的下方。

  • 在「Status Bar」這個區域，可以選擇文件的格式。所以即使副檔名不是「.md」或「.markdown」，也可以改成「Markdown」。

  • 在「Live Preview」的區域右上方，有個設定的圖示，可以設定「Markdown Preview」。
    有兩個下拉選單「Format」和「Theme」。和一個「Checkbox」「Sync scroll position」。
    「Format」通常我都選「GitHub-Flavored(GFM)」。
    「Theme」除了原有的選項。可以參考這裡，去修改，或是增加選項。
    「Sync scroll position」，當勾選時，在「Scroll」編輯區域時，「Live Preview」會跟著「Scroll」。

Console

  • GitHub / Console
  • GitHub / Console Plus

安裝方式

  • 開啟「Extension Manager」對話框。在搜尋控制項輸入「console」。就會看到「Console」和「Console Plus」。兩個可以同時安裝，也可以擇其一安裝。

使用方式

  • 安裝完後，編輯區右方的功能bar，就會出現圖示按鈕。點下就會在編輯區下方出現「Console」的「Panel」。
  • 這個應該是在開發「Extension」的時候有用，若你知道「console.log」，就應該知道這個「Extension」是什麼作用了。

Todo

  • GitHub

安裝方式

  • 開啟「Extension Manager」對話框。在搜尋控制項輸入「todo」。就會看到一筆「Todo」。按下那筆右方的「install」按鈕安裝。

使用方式

  • 安裝完後，編輯區右方的功能bar，就會出現一個「打勾」的「圖示按鈕」。點下就會在編輯區下方出現「Todo」的「Panel」。
  • 只要目前開啟的檔，裡面的註解，有一些關鍵字，例如「// TODO: Yes」。就會列在下方的「Panel」。點下該列。就會將游標移到該行。
  • 關鍵字不是只有「TODO」，還有其他的，請參考「這裡」的說明。
  • 在「Todo」這個「Panel」右上方，還有一個「設定的圖示按鈕」，點下之後，會出現設定的對話視窗。
    可以選擇「Search scope」。有兩個選項「Current file」和「Current project」。
    還有其他的設定，就自行把玩吧！
